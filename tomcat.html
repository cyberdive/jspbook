Setting up Tomcat

First, you will need a Java Development Kit or Java Runtime Environment. For Solaris and NT, these can be downloaded from Sun. JDKs for most versions of Linux are available from Blackdown

Before installing Tomcat, please read the license.

The easiest way to get up and running is to copy the directory tomcat, and all its subdirectories, from the CD to a directory on your hard drive.

Some of the files in this directory have been modified to support the examples from this book. If you would like the 'raw' version of Tomcat, pick a directory in which to install, and unzip tomcat.zip. This will create a tomcat subdirectory containing all the Tomcat code and scripts. Under Unix, make all the scripts runnable with the following command: 

chmod a+x *.sh

To install the examples from this book manually, you can next cd into the Tomcat directory and unzip jspbook.zip.

You are now ready to go! Under Windows run startup.bat, under Solaris, Linux, or other Unix run startup.sh. If you can access this link everything is working properly. If not, please see the README and FAQ files in the tomcat directory.

Running Tomcat with Apache

All the files needed to run Tomcat with Apache under Linux, Solaris and Windows are included on this CD. This is not necessary in order to run the examples from the book, but combining Apache and Tomcat makes for a very powerful platform on which to run JSPs for a real site. Start with the instructions here, all binaries are under the apache directory. As always, check the Jakarta home page for updates.
